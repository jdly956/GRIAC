## 1. S3.19 — re-chunking du xlsx (le cœur)

⚠️ La reprise sur hash ne re-chunke PAS un document en place (algorithme ≠ contenu). Propagation par le geste UI standard :

1. Fiche du document xlsx → **Supprimer** (S3.17) → redéposer le fichier dans son dossier → **Indexer maintenant**.
2. Sur la nouvelle fiche : le document a maintenant **plusieurs chunks-tableaux** (plus un seul géant), chacun ≤ ~800 tokens, et **chaque chunk commence par la ligne d'en-tête + le séparateur** du tableau.
3. Le contenu est compacté : plus de runs d'espaces (les lignes ne font plus 3 600 caractères).
4. Contre-épreuve SQL :
   ```bash
   docker compose -f infra/docker-compose.yml exec postgres psql -U sia -d sia \
     -c "SELECT max(nb_tokens) FROM chunks;"       # attendu : ≤ ~850
   ```
