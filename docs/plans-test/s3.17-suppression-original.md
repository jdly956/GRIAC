## 2. Télécharger l'original (navigateur)

1. « Mes documents » → ouvrir la fiche d'un document **indexé**.
2. Panneau « Actions » → **« Télécharger l'original »** : le navigateur télécharge le fichier **sous son nom d'origine** ; l'ouvrir (Word/PDF) — contenu intact (le proxy est binaire, pas de corruption).
3. Contre-épreuve fichier disparu : sur un pod recréé sans re-dépôt du corpus, le lien répond **404 « Original … absent du corpus de ce pod »** — jamais d'erreur 500.

## 3. Supprimer un document (navigateur)

1. Déposer un document jetable, **Indexer maintenant**, attendre la fin du run — vérifier sur sa fiche : chunks présents et vectorisés.
2. Relever l'état avant suppression :

```bash
cd ~/work/GRIAC/ && source .venv/bin/activate
docker compose -f infra/docker-compose.yml exec postgres psql -U sia -d sia \
  -c "SELECT count(*) FROM chunks WHERE document_chemin = '<chemin du doc>';"
ls ~/work/corpus/<chemin du doc>
```

3. Fiche → « **Supprimer ce document** » → une **confirmation navigateur** s'affiche (annuler une première fois : rien ne se passe) ; confirmer → retour sur « Mes documents », le document a disparu de l'inventaire.
4. Contre-épreuves (attendu : plus rien nulle part) :
   - la requête `chunks` ci-dessus renvoie **0** (cascade FK) ;
   - le fichier source **n'est plus dans `~/work/corpus`** et le dérivé `derived/md/<sha>.md` a disparu ;
   - une question RAG dont la réponse ne vivait que dans ce document → « aucune source récupérable » (anti-invention, le document n'est plus cité) ;
   - **« Indexer maintenant » ne le fait pas revenir** (le source est retiré du corpus — D9).
5. Cas doublon : si un document B était marqué « doublon de » le document supprimé, sa fiche n'affiche plus ce lien (repointé NULL) ; le prochain run de qualification retranche.
