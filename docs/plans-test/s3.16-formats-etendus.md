# Plan de test S3.16 — formats étendus : PowerPoint, Excel, courriels (.eml)

**Environnement cible** : pod Onyxia, branche à jour, api + web relancées (`make pod-up`). Aucune migration.

**Prérequis** : un `.pptx`, un `.xlsx` et un `.eml` réels sous la main (un export « Enregistrer sous > .eml » depuis Outlook/Thunderbird fait l'affaire ; documents **publics ou internes non sensibles uniquement**, pas de données personnelles).

## 1. TNR

```bash
cd ~/work/GRIAC/ && source .venv/bin/activate
make lint && make test        # attendu : 283 verts (279 + 4 nouveaux S3.16)
```

## 2. Upload des nouveaux formats (navigateur)

1. « Mes documents » → le label du champ annonce désormais `docx, pdf, pptx, xlsx, eml, md, txt, odt`.
2. Déposer successivement le `.pptx`, le `.xlsx` et le `.eml` → **trois « Déposé » sans erreur** (avant S3.16 : « Extension refusée »).
3. Contre-test : déposer un `.zip` → toujours refusé (422 rendu en alerte).

## 3. Indexation et fiche document (navigateur)

1. Cliquer **« Indexer maintenant »** ; suivre le run (rafraîchissement 5 s).
   ⚠️ Attendu au premier pptx/xlsx du pod : le nœud parse **télécharge les modèles docling** — le run est plus long, une seule fois.
2. Run terminé : les trois documents passent en statut **parse (indexé)** dans l'inventaire.
3. Ouvrir la **fiche** du `.pptx` : « Voir le résultat du parsing » montre le markdown docling (titres des diapositives, texte des blocs). Idem `.xlsx` : les feuilles rendues en **tableaux markdown**.
4. Ouvrir la fiche du `.eml`. Attendus dans le dérivé :
   - titre = l'**objet** du courriel ;
   - liste `From / To / Cc / Date` ;
   - corps en texte (si le courriel était HTML seul : texte dégradé, aucune balise) ;
   - si pièces jointes : section « **Pièces jointes (non extraites)** » qui les liste — leur contenu n'est PAS indexé (une PJ pertinente se dépose comme document à part).
5. Chunks présents et vectorisés (✅) sur les trois fiches ; les stats de couverture de l'inventaire comptent désormais pptx/xlsx/eml dans les « parsables ».
