## 2. Dépôt avec dossier (navigateur)

1. « Mes documents » → le formulaire de dépôt porte un champ **« Dossier de destination (obligatoire) »** avec l'aide « C'est ce dossier qui est associé à un projet ».
2. Cliquer dans le champ : les **dossiers existants du corpus** sont proposés (datalist).
3. Taper un nom **nouveau** (ex. `projet-pilote`) + déposer un document → « Déposé » ; vérifier sur le pod :
   ```bash
   ls ~/work/corpus/projet-pilote/     # le dossier a été créé, le fichier est dedans
   ```
4. Contre-test : soumettre sans dossier → le navigateur bloque (champ requis) ; en forçant (curl sans champ `dossier`) → **422** avec le détail « obligatoire » :
   ```bash
   cd ~/work/GRIAC/ && source .venv/bin/activate
   curl -s -o /dev/null -w '%{http_code}\n' -X POST localhost:8000/documents/upload \
     -F 'fichier=@/tmp/test.docx'      # attendu : 422
   ```

## 3. La chaîne A6 réparée (le cœur du test)

1. **« Indexer maintenant »** → run terminé, le document est « parse (indexé) » ; sa fiche affiche **« Projet suggéré : projet-pilote »**.
2. Écran **Projets** → ouvrir (ou créer) un projet → le dossier `projet-pilote` **apparaît dans les suggestions** avec son compte de documents → le cocher → enregistrer.
3. Ouvrir une session sur ce projet et poser une question dont la réponse vit dans le document déposé → il est **cité dans les sources** (le filtre projet ne l'exclut plus).
4. Contre-épreuve inverse : une session sur un **autre** projet (dossiers différents) ne cite PAS ce document.
