## 3. Cycle de vie complet d'une session

```bash
# Création (étape 0) — la Feature collée porte déjà une hypothèse : elle entre au registre
curl -sS -X POST http://localhost:8000/workflows -H "Content-Type: application/json" \
  -d '{"feature": "Feature Suivi en ligne.\nFormats acceptés : PDF [HYPOTHÈSE À VALIDER]"}' | python3 -m json.tool
# attendu : 201, etape recuperation_feature, nb_en_attente = 1

# Validation « Oui » (règle 5) -> interview ; l'hypothèse RESTE en_attente (A8)
curl -sS -X POST http://localhost:8000/workflows/1/avancer -d '{"valide": true}' \
  -H "Content-Type: application/json" | python3 -m json.tool

# Validation « Non » -> l'étape ne bouge pas, le commentaire est conservé
curl -sS -X POST http://localhost:8000/workflows/1/avancer \
  -d '{"valide": false, "commentaire": "reformuler la question 2"}' \
  -H "Content-Type: application/json" | python3 -m json.tool
```
