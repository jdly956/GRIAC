## 2. Journalisation des validations (proxy taux d'édition)

Sur une session au navigateur : valider une étape « Oui », puis une autre « Non » avec commentaire. Vérifier :

```bash
kubectl exec -it <pod-postgres> -- psql -U <user> -d <db> \
  -c "SELECT session_id, etape, valide, commentaire FROM workflow_validations ORDER BY id;"
```

Attendu : une ligne par clic, `valide` fidèle au bouton, le commentaire du « Non » présent.

## 3. Notation des stories au navigateur

1. Ouvrir une session dont l'étape de rédaction a produit des US → le panneau **« Noter les stories »** liste les titres extraits (mêmes titres que l'export E5).
2. Noter une story 4/5 avec commentaire → redirection sur la session.
3. Vérifier en base :

```bash
kubectl exec -it <pod-postgres> -- psql -U <user> -d <db> \
  -c "SELECT story_titre, note, commentaire FROM story_feedbacks ORDER BY id;"
```
