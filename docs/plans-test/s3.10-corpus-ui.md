## 2. En réel (navigateur, « Mes documents »)

1. **Déposer** un petit `.md` ou `.docx` → retour sur l'écran, fichier présent dans `$SIA_CORPUS_DIR`. Contre-épreuves : un `.exe` → erreur « extension refusée » lisible ; un nom `../evil.docx` → écrit à plat dans le corpus (pas de traversée).
2. Cliquer **« Indexer maintenant »** → le bouton passe à « Indexation en cours… » (désactivé), la page se rafraîchit toutes les 5 s, le tableau de suivi montre l'avancement **nœud par nœud** (scan → parse → qualify → chunk → embed). Log détaillé : `ingestion-run-<id>.log` (`SIA_LOGS_DIR`).
3. Au terme : statut `termine` (ou `echec_partiel` si OCR requis/quota — la relance reprend sur hash D9), le document déposé apparaît à l'inventaire, la conso embeddings entre dans la jauge S3.11.
4. **Un seul run à la fois** : relancer pendant un run → message « déjà en cours » ; run bloqué (processus tué) → bouton « Marquer en échec » débloque.
